# 운영 매뉴얼

## 운영 URL

```text
https://ruahnote.onrender.com
```

## 주요 경로

| 경로 | 용도 |
|---|---|
| `/` | 스케줄 대시보드 |
| `/notes` | 노트 CRUD |
| `/login` | Supabase Auth |
| `/admin` | 개발자 Admin |
| `/api/health` | 운영 점검 API |

## 기본 점검

1. `/api/health` 응답 확인
2. Render Logs 확인
3. GitHub 최신 commit 확인
4. 로컬에서 `npm.cmd run lint`
5. 로컬에서 `npm.cmd run build`

## Supabase 적용 순서

1. Render와 로컬 `.env.local`에 Supabase 환경변수 등록
2. Direct DB 연결이 실패하면 Session Pooler URL을 `DATABASE_POOLER_URL`로 등록
3. `npm.cmd run db:migrate` 실행
4. `/login`에서 회원가입/로그인 확인
5. `npm.cmd run admin:set -- user@example.com`으로 첫 admin 지정
6. `/admin`에서 admin role 접근 확인
7. `/notes`에서 노트 생성/수정/삭제 확인

## 장애 발생 시

1. `/api/health`에서 누락된 환경변수 확인
2. Render Logs에서 빌드/런타임 오류 확인
3. Supabase Logs에서 Auth/DB/RLS 오류 확인
4. 로컬에서 동일 환경변수로 재현
5. 수정 후 commit/push
