# 운영 매뉴얼

## 운영 URL

```text
https://ruahnote-bp6m.onrender.com
```

## 주요 경로

| 경로 | 용도 |
|---|---|
| `/` | 스케줄 대시보드 |
| `/notes` | 노트 CRUD 및 파일 첨부 |
| `/login` | Supabase Auth |
| `/admin` | 개발자 Admin |
| `/api/health` | 운영 점검 API |

## 기본 점검

1. `/api/health` 응답 확인
2. Render Logs 확인
3. GitHub 최신 commit 확인
4. 로컬에서 `npm.cmd run lint`
5. 로컬에서 `npm.cmd run build`

## Render 환경변수

Render Dashboard에 아래 값을 등록합니다.

```text
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
SUPABASE_SERVICE_ROLE_KEY
DATABASE_POOLER_URL
DATABASE_URL
```

등록 후 `/api/health`의 Supabase 관련 env 값이 `true`인지 확인합니다.

## Supabase 적용 순서

1. 로컬 `.env.local`에 Supabase 환경변수 등록
2. Direct DB 연결이 실패하면 Session Pooler URL을 `DATABASE_POOLER_URL`로 등록
3. `npm.cmd run db:migrate` 실행
4. `/login`에서 회원가입/로그인 확인
5. `npm.cmd run admin:set -- user@example.com`으로 첫 admin 지정
6. `/admin`에서 admin role 접근 확인
7. `/notes`에서 노트 생성/수정/삭제 확인
8. `/notes`에서 파일 업로드 확인

## Render 404 발생 시

1. Render Dashboard에서 latest deploy가 성공인지 확인합니다.
2. Build Command가 `npm ci && npm run build`인지 확인합니다.
3. Start Command가 `npm run render:start`인지 확인합니다.
4. Node version이 20 이상인지 확인합니다.
5. Environment Variables가 등록되어 있는지 확인합니다.
6. Manual Deploy를 실행합니다.
7. `/`, `/api/health`, `/login`, `/notes`, `/admin` 순서로 확인합니다.

## 장애 발생 시

1. `/api/health`에서 누락된 환경변수 확인
2. Render Logs에서 빌드/런타임 오류 확인
3. Supabase Logs에서 Auth/DB/RLS 오류 확인
4. 로컬에서 동일 환경변수로 재현
5. 수정 후 commit/push
