# RuahNote

> AI 기반 기록·노트·녹음·과제·스캔·일정·메일 통합 관리 프로그램

RuahNote는 대학원생, 대학생, 직장인, 연구자, 강사, 프로젝트 실무자가 수업·회의·세미나·업무 기록을 한곳에서 관리할 수 있도록 설계된 통합 생산성 프로그램입니다.

핵심 목표는 **기록 → 정리 → 과제화 → 일정화 → 자료화 → 실행 관리**의 흐름을 하나의 프로그램 안에서 연결하는 것입니다.

---

## 1. 프로그램 핵심 요약

RuahNote는 다음 기능을 중심으로 구현됩니다.

| 구분 | 핵심 기능 |
|---|---|
| 노트 | 카테고리/과목/날짜별 노트 작성, 이미지·파일 첨부, 태그, 검색 |
| 녹음 | 강의·회의 녹음파일 업로드, 녹음 타임라인 기반 필기 연결 |
| AI 강의노트 | 음성 텍스트 변환, 요약, 상세 강의노트 생성, 핵심 키워드 추출 |
| 과제 관리 | 과제 직접 등록, AI 과제 후보 추출, 마감일·알림·완료 보관 |
| 스캔센터 | 책·문서 촬영, 이미지 저장, OCR, PDF 변환, 노트/과제 연결 |
| 캘린더 | 구글 캘린더 조회, 과제 마감일 연동, 주간/월간 일정 관리 |
| 메일 | 다중 구글 메일 계정 조회, 메일 검색, 첨부파일 확인, 메일 작성 |
| 연락처 | 구글 연락처 검색, 교수·조교·팀원에게 바로 메일 작성 |
| 파일함 | 녹음파일, 이미지, PDF, 과제 파일, OCR 텍스트 통합 관리 |
| 설정 | 계정, 알림, OCR, 카메라 품질, AI, 데이터 백업 설정 |

## 2. 기본 데이터 구조

RuahNote의 핵심 구조는 다음과 같습니다.

```text
사용자
└─ 큰 카테고리
   ├─ 2026년 1학기
   │  ├─ 조직신학
   │  │  ├─ 2026-03-05 수업 노트
   │  │  ├─ 2026-03-12 수업 노트
   │  │  └─ 2026-03-19 수업 노트
   │  ├─ 신약개론
   │  │  ├─ 2026-03-06 수업 노트
   │  │  └─ 2026-03-13 수업 노트
   │  └─ 설교학
   │     └─ 2026-03-07 수업 노트
   ├─ 2026년 2학기
   ├─ 회사 프로젝트
   └─ 개인 학습
```

큰 카테고리는 학기, 프로젝트, 회사 업무, 자격증 공부, 개인 연구 등으로 확장 가능합니다.

## 5. 1차 MVP 구현 목표

1차 개발에서는 다음 기능을 우선 구현합니다.

1. 큰 카테고리 생성

6. 이미지 OCR 변환
7. 녹음파일 업로드
8. 음성 텍스트 변환
9. AI 강의노트 생성
10. 과제 등록 및 마감일 관리

12. 과제 완료 보관
13. 스캔 이미지 저장
14. OCR/PDF 변환
15. 구글 캘린더 조회
16. 다중 구글 메일 계정 조회
17. 구글 연락처 검색 및 메일 작성

## 6. 추천 메뉴명

`스캔센터`, `녹음/강의노트`, `과제`, `파일함`은 별도 메뉴로 유지하는 것이 좋습니다.

특히 RuahNote는 단순 노트앱이 아니라 자료 수집, 노트 작성, 강의 정리, 과제 실행까지 연결되는 프로그램이므로 메뉴 구조가 명확해야 합니다.

## 7. 개발 환경 요약

RuahNote는 ChatGPT Codex를 활용한 AI 바이브 코딩 방식으로 구현합니다.

```text
개발 방식: ChatGPT Codex 기반 AI 바이브 코딩
앱 형태: Next.js 웹앱
백엔드: Next.js Route Handler / Server Actions
DB: Supabase PostgreSQL
ORM: Prisma
인증: Supabase Auth + Google OAuth
파일 저장: Supabase Storage
코드 관리: GitHub 필수 사용
배포: Vercel 추천
Firebase: 1차 미사용, 향후 푸시 알림 필요 시 검토
AI: OpenAI API
OCR: Tesseract.js 또는 OCR API
PDF: pdf-lib 또는 jsPDF
```

개발 환경과 Codex 작업 방식은 `04_개발환경_및_AI바이브코딩_기획서.md` 문서에서 별도로 관리합니다.
