# RuahNote 완성까지 작업 스케줄표

## 1. 기준

이 스케줄은 `docs` 폴더의 다음 4개 기획서를 기준으로 작성했다.

- `01_기능구현_총괄기획서.md`
- `02_디자인_기획서.md`
- `03_DB_기획서.md`
- `04_개발환경_및_AI바이브코딩_기획서.md`

현재 프로젝트 상태는 Next.js 기본 생성 직후에 가까우므로, 실질 구현은 0단계부터 시작하는 것으로 본다.

## 2. 완성 기준

RuahNote의 1차 완성 기준은 다음이다.

- Next.js 웹앱 기반의 기본 제품 구조 완성
- Supabase Auth, PostgreSQL, Storage, Prisma 연결
- 큰 카테고리 > 과목/프로젝트 > 날짜별 노트 구조 구현
- 노트 작성, 파일 첨부, 카메라 촬영, OCR, 스캔센터, PDF 변환 구현
- 녹음 업로드, STT, AI 강의노트, 과제 후보 추출 구현
- 과제 관리, 알림 기준, 완료 보관 구현
- Google Calendar, Gmail 다중 계정, Google Contacts 읽기/검색, 메일 작성 구현
- 대시보드에서 일정, 과제, 최근 노트, 스캔/OCR, 메일 상태 확인
- Vercel 배포와 운영 환경변수 정리

## 3. 전체 일정 요약

예상 기간은 16주이다. 혼자 개발하면서 Codex를 적극 활용하는 기준이며, Google OAuth 심사, 외부 API 키 발급, Supabase 정책 설정 지연이 있으면 2~4주가 추가될 수 있다.

| 단계 | 기간 | 핵심 목표 | 산출물 |
|---|---:|---|---|
| 0. 준비 및 기준 정리 | 1주 | 개발 환경, 저장소, 환경변수, 기본 패키지 준비 | GitHub 연결, `.env.example`, 기본 폴더 구조 |
| 1. UI 골격 및 디자인 시스템 | 1주 | Academic Calm Dashboard 레이아웃 구현 | 사이드바, 상단바, 카드, 버튼, 배지 |
| 2. DB/Auth/Storage 기반 | 2주 | Supabase와 Prisma 기반 데이터 구조 구축 | Prisma schema, migration, RLS 초안, Storage bucket |
| 3. 카테고리/과목/노트 MVP | 2주 | 핵심 데이터 계층 CRUD 구현 | 카테고리, 과목, 날짜별 노트 화면/API |
| 4. 노트 에디터와 첨부파일 | 2주 | 문서형 에디터와 파일 관리 구현 | 노트 상세, 이미지/파일 첨부, 태그, 검색 초안 |
| 5. 카메라/OCR/스캔/PDF | 2주 | 이미지 기반 자료 처리 흐름 구현 | 카메라 촬영, OCR, 스캔 폴더, PDF 변환 |
| 6. 과제 관리 | 1.5주 | 과제 등록, 상태, 마감, 완료 보관 구현 | 과제 칸반/리스트, 알림 기준, 연결 자료 |
| 7. 녹음/STT/AI 강의노트 | 2주 | 녹음 업로드와 AI 요약 흐름 구현 | 녹음 업로드, transcript, AI 강의노트, 과제 후보 |
| 8. Google 연동 | 1.5주 | 캘린더, Gmail, 연락처 1차 연동 | OAuth, 캘린더 조회, 메일 조회/작성, 연락처 검색 |
| 9. 통합 대시보드/검색/QA/배포 | 1주 | 전체 흐름 검증 및 배포 | 통합 대시보드, 빌드 통과, Vercel 배포 |

## 5. 기능별 의존관계

| 선행 작업 | 후속 작업 | 이유 |
|---|---|---|
| Supabase Auth | 모든 사용자 데이터 | 사용자별 데이터 분리와 RLS 적용 필요 |
| Prisma 기본 모델 | 카테고리/노트/과제/파일 | 화면보다 데이터 구조가 먼저 안정되어야 함 |
| File/Storage 구조 | 이미지, 녹음, PDF, 메일 첨부 | 모든 자료형이 공통 파일함을 사용함 |
| 노트 CRUD | OCR, 녹음, AI, 과제 후보 | 대부분의 자료가 노트에 연결됨 |
| 스캔 이미지 저장 | OCR/PDF | PDF 변환은 이미지와 OCR 결과에 의존 |
| 녹음 업로드 | STT/AI 강의노트 | AI 강의노트는 transcript가 있어야 생성 가능 |
| Google OAuth | Calendar/Gmail/Contacts | 외부 연동은 계정 연결과 토큰 저장이 선행되어야 함 |

## 9. 주요 리스크

| 리스크 | 영향 | 대응 |
|---|---|---|
| Google OAuth 권한/심사 지연 | Gmail/Contacts 일정 지연 | 로컬 테스트용 계정으로 먼저 개발하고, 배포 전 권한 범위 최소화 |
