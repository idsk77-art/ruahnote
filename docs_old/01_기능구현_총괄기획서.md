# RuahNote 기능구현 중심 총괄 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명

**RuahNote**

### 1.2 한 문장 정의

RuahNote는 수업, 회의, 세미나, 업무 기록을 노트·녹음·스캔·AI 요약·과제·일정·메일과 연결하여 관리하는 AI 기반 기록 운영 시스템이다.

### 1.3 개발 목적

기존 학습 및 업무 기록 방식은 여러 도구에 분산되어 있다.

- 노트 작성: 노션, 삼성노트, 원노트
- 일정 관리: 구글 캘린더
- 녹음: 윈도우 녹음기, 스마트폰 녹음기
- 메일: Gmail
- 파일 관리: 윈도우 폴더, 구글 드라이브
- 강의노트 작성: 수동 정리 또는 AI에 별도 입력

RuahNote는 이 분산된 흐름을 하나로 연결한다. 사용자는 수업이나 회의 중 기록한 데이터가 자동으로 정리되고, 과제와 일정으로 연결되며, 필요한 경우 메일과 연락처까지 이어지는 통합 환경을 사용할 수 있다.

## 4. 데이터 관리 구조

### 4.1 기본 구조

RuahNote의 데이터 구조는 **큰 카테고리 → 하위 분류/과목 → 날짜별 노트**를 중심으로 설계한다.

```text
2026년 1학기
├─ 과목1
│  ├─ 2026-03-05 수업 노트
│  ├─ 2026-03-12 수업 노트
│  └─ 2026-03-19 수업 노트
├─ 과목2
│  ├─ 2026-03-06 수업 노트
│  └─ 2026-03-13 수업 노트
└─ 과목3
   └─ 2026-03-07 수업 노트
```

### 4.2 큰 카테고리 예시

- 2026년 1학기
- 2026년 2학기
- 신학대학원 논문 준비
- 회사 프로젝트
- 자격증 공부
- 독서 기록
- 회의 기록
- 개인 연구

### 4.3 하위 분류 예시

큰 카테고리가 `2026년 1학기`인 경우 하위 분류는 과목이 된다.

- 조직신학
- 신약개론
- 설교학
- 기독교교육
- 상담심리

큰 카테고리가 `회사 프로젝트`인 경우 하위 분류는 프로젝트명이나 부서명이 된다.

- 신규 서비스 기획
- 마케팅 회의
- 고객사 A 프로젝트
- 주간 업무회의

### 4.4 날짜별 노트 구조

각 노트는 날짜를 기준으로 관리한다.

노트 제목 기본값 예시:

- `2026-03-05 조직신학 1주차`
- `2026-03-12 조직신학 2주차`
- `2026-04-02 설교학 발표 준비`
- `2026-05-10 프로젝트 회의록`

노트에는 다음 데이터가 연결될 수 있다.

- 본문 텍스트
- 체크리스트
- 이미지
- 촬영 이미지
- 스캔 이미지
- OCR 텍스트
- 녹음파일
- 음성 변환 텍스트
- AI 강의노트
- 과제
- 일정
- 메일
- 첨부파일

## 5.2 카테고리/과목/노트 관리

### 기능 개요

사용자는 먼저 큰 카테고리를 만들고, 그 안에 과목 또는 프로젝트를 생성한 뒤, 각 과목 하위에 날짜별 노트를 추가한다.

### 주요 기능

- 큰 카테고리 생성
- 큰 카테고리 수정/삭제
- 큰 카테고리 보관
- 하위 분류/과목 생성
- 과목별 색상 또는 아이콘 지정
- 과목별 교수/담당자 정보 등록
- 과목별 기본 요일/시간 등록
- 날짜별 노트 생성
- 날짜별 노트 자동 제목 생성
- 주차 정보 입력
- 노트 검색
- 태그 관리
- 즐겨찾기
- 최근 노트

### 노트 생성 기본 흐름

1. 사용자가 큰 카테고리 선택
2. 하위 과목 선택
3. `새 노트` 클릭
4. 날짜 선택
5. 제목 자동 생성
6. 노트 작성 시작
7. 이미지/녹음/스캔/파일 첨부
8. 저장

## 5.4 카메라 촬영 첨부 기능

### 기능 개요

사용자는 노트 작성 중 태블릿, 스마트폰, 노트북 내장 카메라 또는 웹캠을 통해 사진을 직접 촬영하고 노트에 바로 삽입할 수 있다.

### 주요 기능

- 카메라 권한 요청
- 촬영 화면 실행
- 사진 촬영
- 다시 촬영
- 이미지 미리보기
- 노트에 삽입
- 촬영 이미지 자동 압축
- 촬영 시간 기록
- 녹음 중 촬영 시 타임라인 연결

### 촬영 품질 기준

촬영 이미지는 고화질 보관용이 아니라 웹에서 확인 가능한 중하 품질을 기본으로 한다.

권장 기준:

- 긴 변 1280px~1600px
- JPG 또는 WebP
- 글자 식별 가능
- OCR 가능한 최소 품질 확보
- 빠른 업로드와 저장

## 5.5 이미지 OCR 기능

### 기능 개요

이미지를 첨부하거나 카메라로 촬영했을 때 이미지 안에 텍스트가 있으면 OCR 변환 여부를 묻는다.

### 사용자 안내 문구

`이 이미지에서 텍스트를 추출할까요?`

선택 버튼:

- 텍스트 추출하기
- 이미지만 첨부하기
- 다시 묻지 않기

### OCR 결과 삽입 방식

- 이미지 아래에 텍스트 삽입
- 현재 커서 위치에 삽입
- 별도 OCR 블록으로 삽입
- OCR 결과만 저장

### OCR 활용

- 노트 검색
- 강의노트 생성
- 과제 후보 추출
- 시험 관련 내용 감지
- AI 요약
- 중복 내용 분석

## 5.11 스캔센터

### 기능 개요

스캔센터는 책, 교재, 유인물, 강의자료, 과제 안내문을 카메라로 촬영하여 이미지로 저장하고, 필요 시 OCR 및 PDF로 변환하는 기능이다.

### 스캔 흐름

```text
스캔 폴더 생성
→ 카메라 촬영
→ 이미지 자동 저장
→ 파일명 넘버링
→ OCR 선택
→ OCR 결과 저장
→ PDF 변환 여부 확인
→ PDF 생성
```

### 폴더명 예시

`웨슬리설교_참고자료`

### 파일명 규칙

```text
웨슬리설교_참고자료_001.jpg
웨슬리설교_참고자료_002.jpg
웨슬리설교_참고자료_003.jpg
```

### OCR 메뉴 흐름

1. OCR 메뉴 클릭
2. 스캔 폴더 또는 이미지 선택
3. OCR 적용 이미지 선택
4. 확인
5. OCR 실행
6. 결과 저장
7. PDF 변환 여부 확인

### PDF 메뉴 흐름

1. PDF 변환 메뉴 클릭
2. 이미지 선택
3. 페이지 순서 확인
4. OCR 먼저 적용 여부 확인
5. PDF 변환
6. 노트/과제/파일함에 연결

### PDF 종류

- 이미지 기반 PDF
- OCR 적용 검색 가능 PDF

### 관리 대상

- 이미지
- 카메라 촬영 이미지
- 스캔 이미지
- OCR 텍스트

## 10. 최종 구현 방향

RuahNote는 처음부터 모든 기능을 완성형으로 만들기보다, **데이터 구조와 사용 흐름을 먼저 안정적으로 설계**해야 한다.

가장 중요한 구조는 다음이다.

```text
큰 카테고리 → 과목/프로젝트 → 날짜별 노트 → 연결 자료 → 과제/일정/메일
```

이 구조가 안정적으로 설계되면 이후 AI 기능, OCR, PDF, 구글 연동, 모바일 확장을 자연스럽게 추가할 수 있다.
